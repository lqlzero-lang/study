1 · 文字表示 Text Representation

字符集 Character Set：把每个字符映射到唯一二进制码的规定。计算机存数字，显示硬件再把数字画成字符。
例：'A' → 65 → 01000001

⌨️ 文字编码器（打字看它变二进制）

🧩 解码器（二进制流 → 文字）

先按 7 位一组分开，每组转十进制，再查 ASCII 表。

ASCII vs Unicode

ASCIIUnicode
每字符位数7（或8）16 或更多
字符数量128（或256）100 万+
支持语言只英文所有语言+emoji
文件大小更小更大

Unicode 前 128 个码和 ASCII 一样（向后兼容）。中文、é、emoji 都需要 Unicode。

2 · 声音表示 Sound Representation

声音是空气的连续振动 = 模拟信号 analogue（任意时刻可取任意值）。计算机要把它变成离散的数字，这个过程叫采样 sampling：每隔固定时间量一次振幅，记成二进制数。

🔊 采样可视化（拖滑块看音质怎么变）

蓝线=原始声波，橙点=采样点，绿色阶梯=计算机实际存下来的近似。点越密、级越多 → 越接近原声，但文件越大。

参数 ↑音质文件大小
采样率 sample rate更好更大
采样位深 resolution更好更大
常见参考：电话 8,000 Hz ｜ CD 44,100 Hz ｜ 录音室 96,000 Hz。人耳上限约 20,000 Hz，要准确还原至少采样两倍 → 40,000 Hz，CD 的 44,100 留了余量。采太慢会产生"假音"aliasing。
采样率从 44,100 减半到 22,050，会怎样？音质变差，文件大小减半。

3 · 图像表示 Image Representation

像素 Pixel：数字图像的最小单位，每个像素存一个颜色值。分辨率 resolution=像素总数（宽×高）。色深 colour depth=每个像素用几位表示颜色，n 位 → 2ⁿ 种颜色。

🎨 RGB 调色器（24 位真彩色怎么来的）

24 位真彩色 = 红/绿/蓝 三通道，每通道 8 位(0–255)，3×8=24 位 → 2²⁴ ≈ 1670 万色。拖一拖，看颜色和它的 hex 码（呼应第 1 课！）。

🔍 分辨率对比（像素越多越清晰）

像素越多 → 越清晰 → 文件越大。把滑块往右拖，图会变成马赛克。
色深 4 位，每个像素能有几种颜色？2⁴ = 16 种
为什么提高分辨率会增大文件？像素更多，每个像素都要存颜色数据，总数据量随之增加。
