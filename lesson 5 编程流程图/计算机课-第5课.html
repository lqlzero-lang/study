1 · 什么是算法 What is an Algorithm?

算法 Algorithm：一步一步解决问题的指令。
■ 有限 finite——必须能停下来 ｜ ■ 无歧义 unambiguous——每步只有一个意思 ｜ ■ 有输入有输出 input/output

算法无处不在

任务算法（步骤）
泡一杯茶烧水 → 放茶包 → 倒水 → 等待 → 取出茶包
过马路看左 → 看右 → 安全？ → 走过去
查字典找词翻到中间 → 太前/太后？ → 调整 → 重复

你每天都在用算法，只是没叫它"算法"而已 🙂

算法 ≠ 代码 Algorithm ≠ Code

  • 算法是想法、逻辑；代码 code是用某种语言写出来的实现。
  • 同一个算法可以用 Python、Java 等任何语言写出来。

表达算法的三种方式：① 结构图（拆问题）② 流程图（画流程）③ 伪代码 pseudocode（结构化英语，第 6 课）。

2 · 程序开发周期 PDLC

每个程序都要经过这四个阶段，跳过任何一步都会带来更多 bug 和浪费时间。

1 分析 Analysis

"我们需要什么？"

  • 找出问题：要解决什么？
  • 抽象 abstraction：只关注重要的，忽略无关细节
  • 分解 decomposition：拆成小部分
  • 找出输入、处理、输出、存储
2 设计 Design

"我们要怎么解决？"

  • 写代码之前先规划
  • 用结构图、流程图、伪代码描述
  • 在纸上想清楚，不是在屏幕上
3 编码 Coding

动手写程序

  • 把设计翻译成代码（如 Python）
  • 边写边测，不要拖到最后
  • 随手修 bug
4 测试 Testing

用测试数据检验

  • 用测试数据跑完整程序
  • 每种情况都输出正确吗？
  • 不同类型的测试数据（第 9 课细讲）

🎯 小练习：这是哪个阶段？

先自己想，再点开看答案。

1. 为登录流程画流程图设计 Design（在规划怎么解决）
2. 列出 app 需要向用户收集哪些数据分析 Analysis（在确定需要什么）
3. 用错误密码运行程序看会怎样测试 Testing
4. 写 Python 代码检查密码编码 Coding
5. 判定按钮颜色与问题无关分析 Analysis（这就是抽象 abstraction）

3 · 分解与结构图 Decomposition & Structure Diagrams

分解 Decomposition：把复杂问题拆成更小、可以各自独立解决的子问题。一直拆到每块都简单到能直接解决为止。

结构图把分解过程画出来：树状，上→下是大→小，左→右大致是执行顺序。也叫"自顶向下设计图"。

例：做早餐 Make Breakfast

3 个子系统，每个再拆成 2 个小任务——小到可以直接执行。

  • 网店结账：查看购物车（看商品/改数量/删除）、付款（输卡号/验证）、配送（填地址/选速度/确认）。

4 · 流程图的 6 个符号 Flowchart Symbols

画流程图的规则 Rules

  • 必须有开始 Start 和 结束 Stop。
  • 每个判断 Decision 必须正好有两个出口：Yes/No（或 True/False）。
  • 箭头一定要标方向，别含糊。
  • 用对形状（判断必须是菱形，不是矩形！）——考试用错形状不给分。
  • 保持整洁：流程总体从上到下。
子程序 Subroutine：一段有名字、完成特定任务的代码，可以在主程序里随时"调用 call"。也叫过程 procedure 或函数 function。好处：避免重复代码，让程序更好读、好测试、好维护。

🎯 小练习：该用哪个符号？

1. 开始程序终止符 Terminator（椭圆）
2. 让用户输入名字输入/输出 Input/Output（平行四边形）
3. 计算总价处理 Process（矩形）
4. 检查年龄是否大于 18判断 Decision（菱形）
5. 在屏幕显示结果输入/输出 Input/Output（平行四边形）
6. 调用 "CalculateDiscount" 过程子程序 Subroutine（双边矩形）
7. 结束程序终止符 Terminator（椭圆）

5 · 顺序结构 Sequence

6 · 选择结构 Selection

程序根据条件选择走哪条路，用菱形判断。两个出口 Yes/No，最后再汇合。例子：分数 ≥ 50 就 Pass，否则 Fail。可以改下面的输入分数再跑！

📦 变量 VARIABLES

改好分数，点"单步 ▶"

试试 72 → Pass；再改成 41 → Fail，看菱形走的是 Yes 还是 No。

嵌套选择 Nested Selection

多个判断串起来＝嵌套。比如评等级：≥70 → A，否则 ≥50 → B，否则 Fail。到练习场载入"成绩等级"模板试试看。

7 · 循环（迭代）结构 Iteration

把一段步骤重复执行，直到满足某个条件。流程图里就是一根箭头绕回前面。例子：输出 1 到 5。盯着右边的 Counter 看它怎么一次次变大、又怎么在 Counter>5 时跳出循环。

📦 变量 VARIABLES

点"单步 ▶"，注意箭头绕回去的那一步

计数循环 count-controlled：重复固定次数 ｜ 条件循环 condition-controlled：重复到某条件成立。到练习场试试"累加到 0 为止"。

    🚀 挑战题（自己写，写完到练习场跑）
